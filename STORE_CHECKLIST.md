# Chrome Web Store 上架检查清单

## 一、上架前必查项

### 1. Manifest 合规
- [x] manifest_version: 3
- [x] name: 不含违规关键词
- [x] description: 清晰描述功能
- [x] version: 语义化版本号
- [x] permissions: 最小权限原则
  - [x] 移除不必要的 host_permissions
  - [x] 每个权限都有明确用途
- [x] content_scripts: 匹配规则精确
- [x] 无远程代码加载（MV3 禁止）

### 2. 隐私与安全
- [ ] 隐私政策页面（必须）
- [ ] 数据收集声明
- [ ] 不收集敏感个人信息（密码等明文）
- [ ] HTTPS 通信
- [ ] 不使用 eval() 或 innerHTML（XSS 风险）
- [ ] Content Security Policy 合规

### 3. 功能完整性
- [ ] 核心功能可用
- [ ] 登录流程正常
- [ ] 支付流程正常
- [ ] 错误处理完善
- [ ] 无 console.error 残留

### 4. 素材准备
- [ ] 图标 128x128（PNG，无透明背景）
- [ ] 图标 48x48
- [ ] 图标 16x16
- [ ] 宣传图 440x280（小宣传图）
- [ ] 宣传图 920x680（大宣传图，可选）
- [ ] 截图 1280x800 或 640x400（至少 1 张，最多 5 张）

### 5. 商业合规
- [ ] 开发者账号（一次性 $5 注册费）
- [ ] 隐私政策 URL
- [ ] 服务条款 URL
- [ ] 支持邮箱
- [ ] 如果使用支付：明确标注订阅条款

## 二、提交流程

1. 访问 https://chrome.google.com/webstore/devconsole
2. 点击 "新建项目" 上传 ZIP 包
3. 填写商店信息：
   - 详细描述（支持多语言）
   - 简短描述（132 字符以内）
   - 分类（生产力工具）
   - 语言（中文 + 英文）
4. 填写隐私信息：
   - 是否收集用户数据：是
   - 数据用途说明
   - 隐私政策 URL
5. 设置可见性：
   - 开发者预览 → 测试 → 发布
6. 提交审核（通常 1-3 个工作日）

## 三、常见拒审原因及规避

| 拒审原因 | 规避方法 |
|---------|---------|
| 权限过多 | 只申请必要权限，host_permissions 限制到特定域名 |
| 远程代码执行 | MV3 不允许远程 JS，所有代码打包在扩展内 |
| 隐私政策缺失 | 必须提供可访问的隐私政策 URL |
| 误导性描述 | 截图和描述必须与实际功能一致 |
| 支付未声明 | 如有付费功能，必须在描述中说明 |
| Content Script 过宽 | 匹配规则从 `*://*/*` 收窄到实际需要的域名 |

## 四、ZIP 打包命令

```powershell
# 在 chrome-extension/chrome-extension/ 目录下
Compress-Archive -Path * -DestinationPath ../yuanhai-extension-v2.0.0.zip -Force
```

## 五、审核后注意事项

- 首次审核可能需要 5-7 个工作日
- 后续更新通常 1-2 个工作日
- 审核期间不要修改代码
- 被拒后可修改重新提交
- 保持开发者邮箱畅通
